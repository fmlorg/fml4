ディレクトリ 説明 備考
stable/ 正式リリースにバグ修正だけを施したもの
この stable がお薦めです。 		ファイル名に日付(YYYYMMDD, 例: 20000229) が入っているので新しいものをどうぞ
release/ 正式リリース
current/ 開発版。 新機能は開発版にのみ追加されていきます。
もちろん、利用するに当たってのお薦めは stable です。 ですが、開発サイドとしては 次のリリースのクオリティ向上のため、 current を使って、 テストをしてくださる方が多いことに期待しています。 		ファイル名に日付(YYYYMMDD, 例: 20000229) が入っているので新しいものをどうぞ
daily-snapshots/
  • fml-4.0-current/
  • fml-4.0-stable/
  • fml-devel/
    (次世代研究用 fml)
毎日作るソースのスナップショット。 機械的に毎日作っているものなので、 場合によってはうまく動かないこともあります。 注意: source/ ディレクトリにある tarball は ソース、つまり CVS tree そのものです。 anonymous cvs を使う際の出発点に使って下さい。
utils/ メール関係のユーティリティ
archives/ 過去のスナップショット
current/ も stable/ も daily ではなくある程度区切りのついたところで僕 が作っています。そのためそれなりにちゃんと動くはずです。 一方 daily-snapshots/ は全自動で毎日作っています。daily なので中途半端 なところでパッケージを生成視してしまって失敗している可能性あり。 おっかけをしたい人向きです。

ftp サーバ プライマリは ftp://ftp.fml.org/pub/fml/ セカンダリは ftp://ftp.iij.ad.jp/pub/IIJ/dist/fukachan/fml/ です。

解説 currentもある程度まとまったところで作るのでそれなりに動きますが、たま に失敗していることもあります。テストをしてくれる人が多ければ多いほどリ リースが充実するのでcurrentでひとみ悟空になっていただける、もしくは stable以外にcurrentもちょっとくらい失敗してもいいＭＬで使ってみてもら えると非常にあり難いです。
